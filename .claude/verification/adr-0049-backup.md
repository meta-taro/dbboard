# 検証シート — 論理バックアップ (ダンプ) / ADR-0049

md-business 向けの手動検証シート。dbboard デスクトップの **Backup…**
(論理ダンプ) 機能を対象とする。ダンプのみ (restore は範囲外)。

- **対象機能**: クエリツールバーの **Backup…** → 接続全体を 1 つの `.sql`
  ファイルにダンプ
- **ADR**: ADR-0049 (Accepted 2026-07-22)
- **対象バージョン**: dbboard 0.3.0 系 develop (`feature/adr-0049-logical-dump`)
- **前提**: 接続 id は中立サンプル名 (store-a=D1 / store-b=Aurora DSQL /
  store-c=Supabase)。実店舗名との対応は非公開。SQLite 系の追加確認には
  ローカル Turso (既定の in-memory or ファイル) を使う。

## 判定記号

- ✅ = 期待どおり / ❌ = 不一致 / ➖ = 未実施・対象外
- 「結果」「実測」「備考」列は md-business 実行時に記入する。

## 既知の仕様 (不具合ではない)

- **ダンプのみ**。restore/import は本 ADR の範囲外 (将来 ADR)。
- **Aurora DSQL のダンプは再取り込みを保証しない**。DSQL は FK / sequence /
  `SERIAL` を持たないため該当節は出力されない (degrade by construction)。
- **警告閾値の既定は 500,000 行**。ADR-0050 で設定化済み — メニューバーの
  **Backup** から変更でき、`ui-settings.toml` に保存され再起動後も保持される。
  この検証シートの E5〜E8 は既定値 (500,000) 前提で記載している。
- PK のないテーブルは rowid/ctid/OFFSET フォールバックで読むため、超巨大な
  PK レステーブルではページングコストが上がる (正しさには影響しない)。
